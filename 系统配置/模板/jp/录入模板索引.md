# 录入模板索引

复制下面的模板到对应文件夹后，再补完整字段。`review_stage` 统一使用 `day0 / day1 / day3 / day7 / day14 / day30 / day90 / day180 / mastered`。

## 每日学习清单模板

建议直接复制到当天笔记，不单独新建一套复习条目。

参考文件：[[系统配置/模板/每日学习清单模板]]

适用场景：

- 保存当天实际做了什么
- 记录卡点和断点
- 给明天留一个可直接开做的入口

## 课堂笔记模板

用于当天课堂原始笔记，和每日学习清单分开。

参考文件：[[系统配置/模板/课堂笔记模板]]

默认结构：

- `## 単語`
  - `### 通常語彙`
  - `### 漢字差分`
- `## 文法`
- `## 生活口語`
- `## 間違えた問題`

## 基础词汇模板

完整模板和生成规则见：[[系统配置/模板/单词卡模板]]

适用场景：

- 将已完成重点复习周期的词沉淀到 `学习系统/词库/基础词汇`
- 保留干净 `reading` 与复习用 `accent_display`
- 用常用搭配和基于搭配扩展的例句帮助长期复习
- **多语言兼容**：新卡使用 `pronunciation` 和 `variants` 字段；旧卡的 `reading`+`accent_display` 和 `kanji_diff_pairs` 仍可正常使用

## 课堂词汇模板

完整模板和生成规则见：[[系统配置/模板/单词卡模板]]

适用场景：

- 从课堂笔记、日常学习笔记或图片词汇源创建 `学习系统/词库/重点词汇` 卡

- **多语言兼容**：新卡使用 `pronunciation` 和 `variants` 字段；旧卡的 `reading`+`accent_display` 和 `kanji_diff_pairs` 仍可正常使用

## 生活口语句子卡模板

句卡统一保存在 `学习系统/生活口语/句库/<场景分类>/`，按场景分类建立一层子目录，文件名使用 `<场景>_<核心句或功能>.md`。长篇场景材料、视频笔记和转写附录放在 `学习系统/生活口语/场景指南/`，不要加 `track: survival_speaking`。

现有场景分类：

- `家庭`
- `学校与学习`
- `餐饮`
- `购物`
- `办事与就医`
- `职场`
- `出行`
- `联络与邀约`
- `日常闲聊`

新增句卡优先放入最接近的既有分类；只有确实属于新练习场景时才新增子目录。`scene` frontmatter 仍保留更细粒度的英文 snake_case 值，供面板筛选使用。

Frontmatter 约束：

- `scene` 使用英文 snake_case，例如 `home`、`school`、`small_talk`。
- `speaker_role` 只使用 `self / staff / other`。
- `function` 使用英文 snake_case，不留空。
- `jp_text`、`meaning_zh`、`reply_hint` 必填。
- 有可靠且精确对应的 Shadowing 语音切片时，在正文加入可选的 `## 跟读音频`，直接嵌入原切片；没有可靠切片时省略，不复制音频到句库。

```md
---
track: survival_speaking
status: active
priority: normal
done_today: false
scene:
speaker_role: staff
function:
jp_text:
meaning_zh:
reply_hint:
audio_ref: ""
source_notes: []
first_seen: YYYY-MM-DD
last_seen: YYYY-MM-DD
seen_count: 1
error_count: 0
review_stage: day0
next_review: YYYY-MM-DD
last_reviewed: ""
tags:
  - jp/survival
  - jp/speaking
---

## 什么时候用

说明场景、说话方和使用目的，让这张卡一眼能判断什么时候该用。

## 直接这样说

放核心句、可替换句或同场景变体。

## 对方可能怎么说 / 我怎么回

按 `speaker_role` 选择一个标题保留：自己主动说的卡用 `## 对方可能怎么说`，店员/对方说的卡用 `## 我怎么回`。

## 注意

没有语气、礼貌、唐突风险或适用边界时可以省略。

## 跟读音频

有可靠且精确对应的语音切片时加入嵌入；没有可靠切片时省略整个区块。自然化句卡若引用教材变体，在嵌入前简要标明对应关系。

## 没听懂时

只放最适合这个场景的 1-2 句救场句。
```

## アクセント對比卡规则

アクセント對比卡属于发音练习，不属于普通单词卡。用于同音或近音词需要通过アクセント区分意思、听感或朗读方式的情况。

默认位置：

- `学习系统/发音/アクセント/`

Frontmatter 关键字段：

```yaml
track: pronunciation
item_type: accent_contrast
issue_tags:
  - accent
  - minimal_pair
```

建卡规则：

- 一张卡收 8-12 组，标准按 10 组控制。
- 同一训练目标追加到同一张卡；超过 12 组时开下一张，例如 `アクセント対比_同音異義語_02.md`。
- 反复出错的单组可以拆成高优先级单独卡；原组卡保留简略索引即可。
- 保持在 `学习系统/发音/アクセント/` 下面，和 `学习系统/发音/练习/` 并列；`练习/` 只放句子发音、录音和跟读类练习。
- 如果问题是词义不清，才进入配置中的 focus vocab root；如果问题是アクセント听辨或朗读，放在发音练习。

## 音素對比卡规则

音素對比卡用于清音/浊音、送气、声带振动、口腔动作等最小发音差异。它不放在句子练习目录。

默认位置：

- `学习系统/发音/音素/`

Frontmatter 关键字段：

```yaml
track: pronunciation
item_type: phoneme_contrast
issue_tags:
  - 清浊音混淆
```

建卡规则：

- 规则型、辨别型发音卡放在 `学习系统/发音/音素/`。
- 句子整体语调、停顿、录音复盘放在 `学习系统/发音/练习/`。
- アクセント听辨或朗读问题放在 `学习系统/发音/アクセント/`。
