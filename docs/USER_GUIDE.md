# LingoTrace 用户操作指南

这份指南面向已经会使用 Obsidian，并愿意配合 Codex 或类似 AI agent 维护学习材料的用户。

LingoTrace 的日常入口不是命令行，也不是让用户记内部函数名。你只需要用自然语言说明学习任务，Agent Skill 会判断该做听力笔记、来源笔记、复习卡、口语卡，还是每日复习结算。

## 1. 项目定位

LingoTrace 不是课程、教材内容仓库或通用背单词 App。它提供的是一套外语学习系统骨架：

```text
原始材料
  -> 学习笔记
  -> 复习卡
  -> 定期复习
  -> 输出练习
```

它适合管理：

- 课堂或自学中遇到的词汇、语法和错题。
- 音频、视频材料转成的听力笔记。
- 可复用的表达、生活口语和写作素材。
- 需要间隔复习的学习卡片。

它不提供：

- 私人学习内容。
- 教材音频、视频、PDF 或完整转写稿。
- 通用 ASR 引擎。
- 语言无关的全自动学习系统。

## 2. 准备工作

你需要：

- Obsidian。
- Codex 或兼容的 AI agent。
- 一个自己的私人 Obsidian 日语学习库。
- 可选：ListenKit，用于音视频导入和转写。
- 可选：Git 或 GitHub 下载能力，用于获取 LingoTrace 公共框架。

推荐目录关系：

```text
your-workspace/
  LingoTrace/
  ListenKit/              # 可选，用于听力转写
  your-japanese-library/  # 你的私人 Obsidian 日语学习库
```

公开仓库只保存可复用框架、语言包、模板、测试和说明文档。你的真实学习笔记、音频、每日记录和运行产物应留在自己的学习库里。

## 3. 日常怎么使用

日常使用时，直接对 AI agent 说学习目标。不要把内部路径、函数名或实现细节当作用户指令。

可以这样说：

- “请把这段音频做成精听稿。”
- “帮我把这篇材料整理成日语学习笔记。”
- “把这个词加入复习。”
- “这句话很实用，帮我做成口语卡。”
- “今天复习结束了，帮我结算。”

Agent Skill 会根据请求选择合适的能力，并保存到你的日语学习库。缺少音频、来源、日期或必要上下文时，agent 会先向你确认。

上面的句子只是示例，不是固定提示词。Agent 会先识别你的真实学习意图，再选择对应能力。“更新总训练表”和“请更新总训练表”是明确的每日复习结算请求。只有“处理一下总训练表”“总训练表有点问题”这类未说明是结算还是视图维护的表达，agent 才应先问一句确认问题。

## 4. 五个学习任务

### 听力笔记

用于把本地音频或媒体 URL 变成泛听笔记或精听稿。

适合这样说：

```text
请把 23.mp3 做成精听稿。
```

精听稿应包含真实音频切片，不能只生成空的 Markdown 结构。系统会先检查音频和切片工具；如果需要修改已有笔记，会先停下来确认，不会覆盖你已经手工整理过的笔记。

### 来源笔记

用于把视频、文章、转写稿或混合材料整理成可追溯的学习笔记。

适合这样说：

```text
帮我把这篇采访整理成日语学习笔记。
```

来源笔记的重点不是简单总结，而是保留材料来源、学习价值和后续复习卡的出处。需要做复习卡时，先完成来源笔记，再从里面挑选具体学习点。

### 复习材料

用于维护词汇卡、语法卡、发音卡和错题卡。

适合这样说：

```text
把「相撲」加入复习。
```

Agent 会先查重，再决定是否新建。涉及合并、移动、覆盖或复习状态变化时，会先让我确认将要新增或修改的内容。

### 生活口语卡

用于保存你确认过、以后真的想说出口的短句。

适合这样说：

```text
这句话很实用，帮我做成口语卡。
```

不要把未经确认的 ASR 文本、教材操练句或不自然表达直接放入口语库。口语卡应短小、场景明确、方便复述。

### 每日复习结算

用于结束当天复习并更新下一次复习日期。

适合这样说：

```text
今天复习结束了，帮我结算。
```

明确的复习结算请求不需要二次确认。Agent 会先做内部预览检查；如果没有错误，就直接保存结算结果，再做一次预览确认没有剩余待结算卡片。完成后会报告有多少卡片前进到下一阶段、有多少卡片完成，以及是否有阻塞项。

## 5. 听力链与 ListenKit

如果你要处理音频或视频，推荐把 ListenKit 放在 LingoTrace 旁边。ListenKit 负责音视频导入、转写和切片，LingoTrace 负责把结果整理成学习笔记和复习材料。

当你说“请把这段音频做成精听稿”时，agent 应该完成完整体验：

1. 检查音频或 URL 是否可用。
2. 检查转写和切片工具。
3. 生成或复用转写和切片证据。
4. 生成精听稿。
5. 保存到你的日语学习库。
6. 告诉你生成了哪篇笔记、包含多少个精听片段，以及是否需要人工复核。

正常学习时，你不需要手工准备内部数据结构。工具缺失或材料不足时，agent 会说明缺什么，并在改动文件前停止。

## 6. 公开模板和私人内容

Japanese language pack 提供可复用模板和默认视图，例如总训练面板模板。你可以把这些模板复制到自己的学习库后再按个人习惯调整。

公共仓库可以迭代：

- 可复用模板。
- 默认视图。
- Agent Skill 说明。
- 语言包规则。
- 测试和安全检查。

公共仓库不应包含：

- 私人日记和学习记录。
- 课堂原始笔记。
- 教材音频、视频、PDF、截图。
- 商业教材完整转写稿。
- Obsidian 本地工作区状态。
- 临时转写产物、缓存和媒体文件。

## 7. 如何扩展到英语或其他外语

这个项目可以迁移到其他外语，但迁移的是系统思想，不是直接复用日语语言包。

可以复用：

- Obsidian 管理方式。
- 来源笔记到复习卡再到复习结算的流程。
- 自然语言 + Agent Skill 的协作方式。
- 公开仓库和私人学习库分离的安全边界。
- 听力材料、口语句库、错题卡等模块划分。

需要改造：

- 语言包和 Agent Skill。
- 词汇、语法、发音和口语卡模板。
- 听力转写语言参数和后处理规则。
- 示例句、标签、路径角色和默认视图。

建议先选择一个小范围试点，例如“英语来源笔记 + 词汇卡”，不要一次性把所有日语规则改成通用规则。

## 8. 推荐上手路径

第一次使用时，建议按这个顺序：

1. 准备一个自己的 Obsidian 日语学习库。
2. 安装或打开能读取 LingoTrace Agent Skill 的 AI agent。
3. 选择一个小材料，例如一段短音频或一页自学笔记。
4. 用自然语言要求 agent 生成听力笔记或来源笔记。
5. 从笔记里人工确认几个词汇、语法点或口语句。
6. 让 agent 建少量复习卡。
7. 当天复习后，让 agent 做一次结算。

这套系统的重点不是一次性自动生成很多内容，而是把学习材料稳定地沉淀成可复习、可追溯、可长期维护的结构。
